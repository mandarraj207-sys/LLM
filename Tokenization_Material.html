Python & Regex for Tokenisation

An orchestrated pedagogical sequence  ·  IIT Hyderabad · Summer Project 2026

What is tokenisation? Before an LLM can learn from text, it must convert raw characters into discrete units called tokens. This page explains every Python command used in Chapter 2 to build that pipeline — in the order that makes the most sense for learning, not the order they appear in the notebook.
1

Reading a file

f.read()

Reads the entire contents of an open file and returns it as one long string. The f is just the variable name for the file object — it could be anything.

with open("the-verdict.txt", "r", encoding="utf-8") as f:
    raw_text = f.read()
"r" → open in read mode (not write) encoding="utf-8" → interpret the bytes as UTF-8 text f.read() → pulls all characters into a single string
Output
Total no of characters: 20481
"I HAD always thought Jack Gisburn rather a cheap genius..."
The with open(…) as f: block automatically closes the file when the indented block ends — safer than calling f.close() yourself.
2

Splitting on whitespace

re.split(r'(\s)', text)

re.split breaks a string wherever a pattern matches. Here the pattern is \s, which means any whitespace character (space, tab, newline).

re.split(r'(\s)', text)
r'...' → raw string: backslashes are NOT treated as Python escape characters \s → regex shorthand for any whitespace (space, tab, \n, \r) ( ) → the parentheses KEEP the delimiter in the output list
Input → Output
Input: "hello world! This, is: a test."
Output: ['hello', ' ', 'world!', ' ', 'This,', ' ', 'is:', ' ', 'a', ' ', 'test.']
Without the parentheses — r'\s' — the spaces would be discarded. The parentheses capture them so you can inspect where splits happened.
3

Stripping whitespace

item.strip()

Removes leading and trailing whitespace (spaces, newlines, tabs) from a string. It does not touch whitespace in the middle.

"  hello  ".strip() → "hello"   (outer spaces removed)
"  ".strip() → ""   (empty string — this is falsy in Python)
"hello world".strip() → "hello world"   (middle space untouched)
An empty string "" evaluates to False in Python. This becomes the filter mechanism in the list comprehension — Step 6.
4

Adding punctuation to the split

re.split(r'([,.]|\s)', text)

Now we split on commas, full stops, or whitespace. The | means "or" — the pattern matches whichever alternative it finds first.

re.split(r'([,.]|\s)', text)
[,.] → character class: matches either , or . |   → OR \s → any whitespace ( ) → keep matched delimiters in result
Input → Output
Input: "hello world! This, is: a test."
Output: ['hello', ' ', 'world!', ' ', 'This', ',', '', ' ', 'is:', ' ', 'a', ' ', 'test', '.', '']
Notice the empty strings '' — they appear when a comma is immediately followed by a space: splitting on both leaves nothing between them. These must be cleaned up.
5

The full tokeniser pattern

re.split(r'([,.:;?_!"()\']\|--|\s)', text)

The complete punctuation-aware split used in Chapter 2. It handles all common punctuation marks plus the double-dash -- as a single token.

re.split(r'([,.:;?_!"()\']\|--|\s)', text)
[,.:;?_!"()\' ] → all common punctuation. The \' is an escaped apostrophe |                → OR --              → literal double-dash, matched as one unit (em-dash substitute) |                → OR \s              → any whitespace
Input → Output
Input: "hello world! This, is -- a test."
Output: ['hello', 'world', '!', 'This', ',', 'is', '--', 'a', 'test', '.']
Order matters: -- must appear before \s in the pattern. Regex tries alternatives left-to-right — if whitespace matched first, the double-dash might never be captured as a single unit.
6

List comprehension — clean the result

result = [item.strip() for item in result if item.strip()]

One line that does two things simultaneously: strips whitespace from every item, and discards any item that becomes empty after stripping.

result = [item.strip()  for item in result  if item.strip()]
item.strip() (before for) → what to PUT in the new list for item in result → loop over every element; this CREATES the variable item if item.strip() → only include if strip() returns a non-empty (truthy) string
Before → After
Before: ['hello', ' ', 'world!', ' ', 'This', ',', '', ' ', 'is:', ' ', 'a', ' ', 'test', '.', '']
After: ['hello', 'world!', 'This', ',', 'is:', 'a', 'test', '.']
The if item.strip() at the end is the filter. An empty string "" and a lone space " " both evaluate to False, so they are excluded. This is why understanding strip()'s falsy behaviour in Step 3 matters here.
?

Deep dive — where is item defined?

Student question

I don't see item defined anywhere. Where does it come from?

item is defined right there inside the comprehension — by the for part. It does not exist before the loop starts. The for item in result clause is what creates the variable item and reassigns it on every iteration. There is no separate definition line above it.

The confusion arises because a list comprehension is really a regular for-loop written in compressed form. Here is the same logic written both ways — they produce identical results:

Regular for-loop
# item is created by the for line
new_list = []
for item in result:
    if item.strip():
        new_list.append(
            item.strip()
        )
List comprehension
# compressed into one line



result = [
    item.strip()
    for item in result
    if item.strip()
]

They are 100% identical in what they do. The only difference is that in the comprehension syntax, Python requires you to write the output expression first — even though logically the loop runs first.

Reading order vs writing order
① for item in result
→
② if item.strip()
→
③ collect item.strip()

The trick is to start reading from the middle, not the left:

"For each item in result — if item.strip() is non-empty — give me item.strip()"
Why write it as a comprehension at all? — Three reasons: it is faster than a regular loop (Python optimises it internally), it is shorter, and once you get used to reading them, it is actually clearer. Most Python programmers strongly prefer comprehensions for simple filter-and-transform operations like this one.
